                <div class="highlight"><pre><div class='line' id='LC1'>*surround.txt*  Plugin for deleting, changing, and adding &quot;surroundings&quot;</div><div class='line' id='LC2'><br/></div><div class='line' id='LC3'>Author:  Tim Pope &lt;http://tpo.pe/&gt;</div><div class='line' id='LC4'>License: Same terms as Vim itself (see |license|)</div><div class='line' id='LC5'><br/></div><div class='line' id='LC6'>This plugin is only available if &#39;compatible&#39; is not set.</div><div class='line' id='LC7'><br/></div><div class='line' id='LC8'>INTRODUCTION                                    *surround*</div><div class='line' id='LC9'><br/></div><div class='line' id='LC10'>This plugin is a tool for dealing with pairs of &quot;surroundings.&quot;  Examples</div><div class='line' id='LC11'>of surroundings include parentheses, quotes, and HTML tags.  They are</div><div class='line' id='LC12'>closely related to what Vim refers to as |text-objects|.  Provided</div><div class='line' id='LC13'>are mappings to allow for removing, changing, and adding surroundings.</div><div class='line' id='LC14'><br/></div><div class='line' id='LC15'>Details follow on the exact semantics, but first, consider the following</div><div class='line' id='LC16'>examples.  An asterisk (*) is used to denote the cursor position.</div><div class='line' id='LC17'><br/></div><div class='line' id='LC18'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC19'>&nbsp;&nbsp;&quot;Hello *world!&quot;           ds&quot;         Hello world!</div><div class='line' id='LC20'>&nbsp;&nbsp;[123+4*56]/2              cs])        (123+456)/2</div><div class='line' id='LC21'>&nbsp;&nbsp;&quot;Look ma, I&#39;m *HTML!&quot;     cs&quot;&lt;q&gt;      &lt;q&gt;Look ma, I&#39;m HTML!&lt;/q&gt;</div><div class='line' id='LC22'>&nbsp;&nbsp;if *x&gt;3 {                 ysW(        if ( x&gt;3 ) {</div><div class='line' id='LC23'>&nbsp;&nbsp;my $str = *whee!;         vlllls&#39;     my $str = &#39;whee!&#39;;</div><div class='line' id='LC24'><br/></div><div class='line' id='LC25'>While a few features of this plugin will work in older versions of Vim,</div><div class='line' id='LC26'>Vim 7 is recommended for full functionality.</div><div class='line' id='LC27'><br/></div><div class='line' id='LC28'>MAPPINGS                                        *surround-mappings*</div><div class='line' id='LC29'><br/></div><div class='line' id='LC30'>Delete surroundings is *ds* .  The next character given determines the target</div><div class='line' id='LC31'>to delete.  The exact nature of the target is explained in |surround-targets|</div><div class='line' id='LC32'>but essentially it is the last character of a |text-object|.  This mapping</div><div class='line' id='LC33'>deletes the difference between the &quot;i&quot;nner object and &quot;a&quot;n object.  This is</div><div class='line' id='LC34'>easiest to understand with some examples:</div><div class='line' id='LC35'><br/></div><div class='line' id='LC36'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC37'>&nbsp;&nbsp;&quot;Hello *world!&quot;           ds&quot;         Hello world!</div><div class='line' id='LC38'>&nbsp;&nbsp;(123+4*56)/2              ds)         123+456/2</div><div class='line' id='LC39'>&nbsp;&nbsp;&lt;div&gt;Yo!*&lt;/div&gt;           dst         Yo!</div><div class='line' id='LC40'><br/></div><div class='line' id='LC41'>Change surroundings is *cs* .  It takes two arguments, a target like with</div><div class='line' id='LC42'>|ds|, and a replacement.  Details about the second argument can be found</div><div class='line' id='LC43'>below in |surround-replacements|.  Once again, examples are in order.</div><div class='line' id='LC44'><br/></div><div class='line' id='LC45'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC46'>&nbsp;&nbsp;&quot;Hello *world!&quot;           cs&quot;&#39;        &#39;Hello world!&#39;</div><div class='line' id='LC47'>&nbsp;&nbsp;&quot;Hello *world!&quot;           cs&quot;&lt;q&gt;      &lt;q&gt;Hello world!&lt;/q&gt;</div><div class='line' id='LC48'>&nbsp;&nbsp;(123+4*56)/2              cs)]        [123+456]/2</div><div class='line' id='LC49'>&nbsp;&nbsp;(123+4*56)/2              cs)[        [ 123+456 ]/2</div><div class='line' id='LC50'>&nbsp;&nbsp;&lt;div&gt;Yo!*&lt;/div&gt;           cst&lt;p&gt;      &lt;p&gt;Yo!&lt;/p&gt;</div><div class='line' id='LC51'><br/></div><div class='line' id='LC52'>*ys* takes a valid Vim motion or text object as the first object, and wraps</div><div class='line' id='LC53'>it using the second argument as with |cs|.  (It&#39;s a stretch, but a good</div><div class='line' id='LC54'>mnemonic for &quot;ys&quot; is &quot;you surround&quot;.)</div><div class='line' id='LC55'><br/></div><div class='line' id='LC56'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC57'>&nbsp;&nbsp;Hello w*orld!             ysiw)       Hello (world)!</div><div class='line' id='LC58'><br/></div><div class='line' id='LC59'>As a special case, *yss* operates on the current line, ignoring leading</div><div class='line' id='LC60'>whitespace.</div><div class='line' id='LC61'><br/></div><div class='line' id='LC62'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC63'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Hello w*orld!         yssB            {Hello world!}</div><div class='line' id='LC64'><br/></div><div class='line' id='LC65'>There is also *yS* and *ySS* which indent the surrounded text and place it</div><div class='line' id='LC66'>on a line of its own.</div><div class='line' id='LC67'><br/></div><div class='line' id='LC68'>In visual mode, a simple &quot;s&quot; with an argument wraps the selection.  This is</div><div class='line' id='LC69'>referred to as the *vS* mapping, although ordinarily there will be</div><div class='line' id='LC70'>additional keystrokes between the v and s.  In linewise visual mode, the</div><div class='line' id='LC71'>surroundings are placed on separate lines and indented.  In blockwise visual</div><div class='line' id='LC72'>mode, each line is surrounded.</div><div class='line' id='LC73'><br/></div><div class='line' id='LC74'>A &quot;gS&quot; in visual mode, known as *vgS* , behaves similarly.  In linewise visual</div><div class='line' id='LC75'>mode, the automatic indenting is surpressed.  In blockwise visual mode, this</div><div class='line' id='LC76'>enables surrounding past the end of the like with &#39;virtualedit&#39; set (there</div><div class='line' id='LC77'>seems to be no way in Vim Script to differentiate between a jagged end of line</div><div class='line' id='LC78'>selection and a virtual block selected past the end of the line, so two maps</div><div class='line' id='LC79'>were needed).</div><div class='line' id='LC80'><br/></div><div class='line' id='LC81'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*i_CTRL-G_s* *i_CTRL-G_S*</div><div class='line' id='LC82'>Finally, there is an experimental insert mode mapping on &lt;C-G&gt;s and &lt;C-S&gt;.</div><div class='line' id='LC83'>Beware that the latter won&#39;t work on terminals with flow control (if you</div><div class='line' id='LC84'>accidentally freeze your terminal, use &lt;C-Q&gt; to unfreeze it).  The mapping</div><div class='line' id='LC85'>inserts the specified surroundings and puts the cursor between them.  If,</div><div class='line' id='LC86'>immediately after the mapping and before the replacement, a second &lt;C-S&gt; or</div><div class='line' id='LC87'>carriage return is pressed, the prefix, cursor, and suffix will be placed on</div><div class='line' id='LC88'>three separate lines.  &lt;C-G&gt;S (not &lt;C-G&gt;s) also exhibits this behavior.</div><div class='line' id='LC89'><br/></div><div class='line' id='LC90'>TARGETS                                         *surround-targets*</div><div class='line' id='LC91'><br/></div><div class='line' id='LC92'>The |ds| and |cs| commands both take a target as their first argument.  The</div><div class='line' id='LC93'>possible targets are based closely on the |text-objects| provided by Vim.</div><div class='line' id='LC94'>All targets are currently just one character.</div><div class='line' id='LC95'><br/></div><div class='line' id='LC96'>Eight punctuation marks, (, ), {, }, [, ], &lt;, and &gt;, represent themselves</div><div class='line' id='LC97'>and their counterparts.  If the opening mark is used, contained whitespace is</div><div class='line' id='LC98'>also trimmed.  The targets b, B, r, and a are aliases for ), }, ], and &gt; </div><div class='line' id='LC99'>(the first two mirror Vim; the second two are completely arbitrary and</div><div class='line' id='LC100'>subject to change).</div><div class='line' id='LC101'><br/></div><div class='line' id='LC102'>Three quote marks, &#39;, &quot;, `, represent themselves, in pairs.  They are only</div><div class='line' id='LC103'>searched for on the current line.</div><div class='line' id='LC104'><br/></div><div class='line' id='LC105'>A t is a pair of HTML or XML tags.  See |tag-blocks| for details.  Remember</div><div class='line' id='LC106'>that you can specify a numerical argument if you want to get to a tag other</div><div class='line' id='LC107'>than the innermost one.</div><div class='line' id='LC108'><br/></div><div class='line' id='LC109'>The letters w, W, and s correspond to a |word|, a |WORD|, and a |sentence|,</div><div class='line' id='LC110'>respectively.  These are special in that they have nothing to delete, and</div><div class='line' id='LC111'>used with |ds| they are a no-op.  With |cs|, one could consider them a</div><div class='line' id='LC112'>slight shortcut for ysi (cswb == ysiwb, more or less).</div><div class='line' id='LC113'><br/></div><div class='line' id='LC114'>A p represents a |paragraph|.  This behaves similarly to w, W, and s above;</div><div class='line' id='LC115'>however, newlines are sometimes added and/or removed.</div><div class='line' id='LC116'><br/></div><div class='line' id='LC117'>REPLACEMENTS                                    *surround-replacements*</div><div class='line' id='LC118'><br/></div><div class='line' id='LC119'>A replacement argument is a single character, and is required by |cs|, |ys|,</div><div class='line' id='LC120'>and |vs|.  Undefined replacement characters (with the exception of alphabetic</div><div class='line' id='LC121'>characters) default to placing themselves at the beginning and end of the</div><div class='line' id='LC122'>destination, which can be useful for characters like / and |.</div><div class='line' id='LC123'><br/></div><div class='line' id='LC124'>If either ), }, ], or &gt; is used, the text is wrapped in the appropriate pair</div><div class='line' id='LC125'>of characters.  Similar behavior can be found with (, {, and [ (but not &lt;),</div><div class='line' id='LC126'>which append an additional space to the inside.  Like with the targets above,</div><div class='line' id='LC127'>b, B, r, and a are aliases for ), }, ], and &gt;.  To fulfill the common need for</div><div class='line' id='LC128'>code blocks in C-style languages, &lt;C-}&gt; (which is really &lt;C-]&gt;) adds braces on</div><div class='line' id='LC129'>lines separate from the content.</div><div class='line' id='LC130'><br/></div><div class='line' id='LC131'>If t or &lt; is used, Vim prompts for an HTML/XML tag to insert.  You may specify</div><div class='line' id='LC132'>attributes here and they will be stripped from the closing tag.  End your</div><div class='line' id='LC133'>input by pressing &lt;CR&gt; or &gt;.  If &lt;C-T&gt; is used, the tags will appear on lines</div><div class='line' id='LC134'>by themselves.</div><div class='line' id='LC135'><br/></div><div class='line' id='LC136'>A deprecated replacement of a LaTeX environment is provided on \ and l.  The</div><div class='line' id='LC137'>name of the environment and any arguments will be input from a prompt.  This</div><div class='line' id='LC138'>will be removed once a more fully functional customization system is</div><div class='line' id='LC139'>implemented.  The following shows the resulting environment from</div><div class='line' id='LC140'>csp\tabular}{lc&lt;CR&gt;</div><div class='line' id='LC141'>&gt;</div><div class='line' id='LC142'>&nbsp;&nbsp;\begin{tabular}{lc}</div><div class='line' id='LC143'>&nbsp;&nbsp;\end{tabular}</div><div class='line' id='LC144'>&lt;</div><div class='line' id='LC145'>CUSTOMIZING                                     *surround-customizing*</div><div class='line' id='LC146'><br/></div><div class='line' id='LC147'>The following adds a potential replacement on &quot;-&quot; (ASCII 45) in PHP files.</div><div class='line' id='LC148'>(To determine the ASCII code to use, :echo char2nr(&quot;-&quot;)).  The carriage</div><div class='line' id='LC149'>return will be replaced by the original text.</div><div class='line' id='LC150'>&gt;</div><div class='line' id='LC151'>&nbsp;&nbsp;autocmd FileType php let b:surround_45 = &quot;&lt;?php \r ?&gt;&quot;</div><div class='line' id='LC152'>&lt;</div><div class='line' id='LC153'>This can be used in a PHP file as in the following example.</div><div class='line' id='LC154'><br/></div><div class='line' id='LC155'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC156'>&nbsp;&nbsp;print &quot;Hello *world!&quot;     yss-        &lt;?php print &quot;Hello world!&quot; ?&gt;</div><div class='line' id='LC157'><br/></div><div class='line' id='LC158'>Additionally, one can use a global variable for globally available</div><div class='line' id='LC159'>replacements.</div><div class='line' id='LC160'>&gt;</div><div class='line' id='LC161'>&nbsp;&nbsp;let g:surround_45 = &quot;&lt;% \r %&gt;&quot;</div><div class='line' id='LC162'>&nbsp;&nbsp;let g:surround_61 = &quot;&lt;%= \r %&gt;&quot;</div><div class='line' id='LC163'>&lt;</div><div class='line' id='LC164'>Advanced, experimental, and subject to change:  One can also prompt for</div><div class='line' id='LC165'>replacement text.  The syntax for this is to surround the replacement in pairs</div><div class='line' id='LC166'>of low numbered control characters.  If this sounds confusing, that&#39;s because</div><div class='line' id='LC167'>it is (but it makes the parsing easy).  Consider the following example for a</div><div class='line' id='LC168'>LaTeX environment on the &quot;l&quot; replacement.</div><div class='line' id='LC169'>&gt;</div><div class='line' id='LC170'>&nbsp;&nbsp;let g:surround_108 = &quot;\\begin{\1environment: \1}\r\\end{\1\1}&quot;</div><div class='line' id='LC171'>&lt;</div><div class='line' id='LC172'>When this replacement is used,  the user is prompted with an &quot;environment: &quot;</div><div class='line' id='LC173'>prompt for input.  This input is inserted between each set of \1&#39;s.</div><div class='line' id='LC174'>Additional inputs up to \7 can be used.</div><div class='line' id='LC175'><br/></div><div class='line' id='LC176'>Furthermore, one can specify a regular expression substitution to apply.</div><div class='line' id='LC177'>&gt;</div><div class='line' id='LC178'>&nbsp;&nbsp;let g:surround_108 = &quot;\\begin{\1environment: \1}\r\\end{\1\r}.*\r\1}&quot;</div><div class='line' id='LC179'>&lt;</div><div class='line' id='LC180'>This will remove anything after the first } in the input when the text is</div><div class='line' id='LC181'>placed within the \end{} slot.  The first \r marks where the pattern begins,</div><div class='line' id='LC182'>and the second where the replacement text begins.</div><div class='line' id='LC183'><br/></div><div class='line' id='LC184'>Here&#39;s a second example for creating an HTML &lt;div&gt;.  The substitution cleverly</div><div class='line' id='LC185'>prompts for an id, but only adds id=&quot;&quot; if it is non-blank.  You may have to</div><div class='line' id='LC186'>read this one a few times slowly before you understand it.</div><div class='line' id='LC187'>&gt;</div><div class='line' id='LC188'>&nbsp;&nbsp;let g:surround_{char2nr(&quot;d&quot;)} = &quot;&lt;div\1id: \r..*\r id=\&quot;&amp;\&quot;\1&gt;\r&lt;/div&gt;&quot;</div><div class='line' id='LC189'>&lt;</div><div class='line' id='LC190'>Inputting text replacements is a proof of concept at this point. The ugly,</div><div class='line' id='LC191'>unintuitive interface and the brevity of the documentation reflect this.</div><div class='line' id='LC192'><br/></div><div class='line' id='LC193'>Finally, It is possible to always append a string to surroundings in insert</div><div class='line' id='LC194'>mode (and only insert mode).  This is useful with certain plugins and mappings</div><div class='line' id='LC195'>that allow you to jump to such markings.</div><div class='line' id='LC196'>&gt;</div><div class='line' id='LC197'>&nbsp;&nbsp;let g:surround_insert_tail = &quot;&lt;++&gt;&quot;</div><div class='line' id='LC198'>&lt;</div><div class='line' id='LC199'>ISSUES                                          *surround-issues*</div><div class='line' id='LC200'><br/></div><div class='line' id='LC201'>Vim could potentially get confused when deleting/changing occurs at the very</div><div class='line' id='LC202'>end of the line.  Please report any repeatable instances of this.</div><div class='line' id='LC203'><br/></div><div class='line' id='LC204'>Do we need to use |inputsave()|/|inputrestore()| with the tag replacement?</div><div class='line' id='LC205'><br/></div><div class='line' id='LC206'>Indenting is handled haphazardly.   <h2>Markdown Cheat Sheet</h2>

  <div class="cheatsheet-content">

  <div class="mod">
    <div class="col">
      <h3>Format Text</h3>
      <p>Headers</p>
      <pre>
# This is an &lt;h1&gt; tag
## This is an &lt;h2&gt; tag
###### This is an &lt;h6&gt; tag</pre>
     <p>Text styles</p>
     <pre>
*This text will be italic*
_This will also be italic_
**This text will be bold**
__This will also be bold__

*You **can** combine them*
</pre>
    </div>
    <div class="col">
      <h3>Lists</h3>
      <p>Unordered</p>
      <pre>
* Item 1
* Item 2
  * Item 2a
  * Item 2b</pre>
     <p>Ordered</p>
     <pre>
1. Item 1
2. Item 2
3. Item 3
   * Item 3a
   * Item 3b</pre>
    </div>
    <div class="col">
      <h3>Miscellaneous</h3>
      <p>Images</p>
      <pre>
![GitHub Logo](/images/logo.png)
Format: ![Alt Text](url)
</pre>
     <p>Links</p>
     <pre>
http://github.com - automatic!
[GitHub](http://github.com)</pre>
<p>Blockquotes</p>
     <pre>
As Kanye West said:
> We're living the future so
> the present is our past.
</pre>
    </div>
